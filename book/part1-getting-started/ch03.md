## 3.1 Slash Commands 核心指令全景

Slash Commands 是 Claude Code 的专属快捷指令体系，所有指令以 `/` 开头。它的核心价值是**替代冗长的自然语言描述**，将高频、固定的操作固化为一键指令，大幅简化交互流程、减少重复沟通，是提升 AI 编程效率的基础核心能力。

以下为日常开发中使用率最高、必备精通的八大核心指令，覆盖 99% 的常规操作场景：

**1. `/help` 帮助查询**

查看 Claude Code 全部可用指令与功能说明。新手无需死记所有命令，随时输入即可调取完整指令清单。首次启动 Claude Code 时，工具也会主动推荐该指令，是入门必备基础命令。

**2. `/clear` 清空会话**

清空当前全部对话历史、重置会话上下文，相当于”重启一局全新对话”。适用于单个功能开发完成、切换完全不相关的新任务场景，彻底避免历史对话信息干扰新需求，同时重置 Token 消耗统计。

**3. `/compact` 上下文压缩**

智能压缩超长对话上下文，对历史沟通内容进行摘要留存，释放可用 Token 空间。与 `/clear` 最大的区别是：**保留核心记忆，不丢失历史沟通逻辑**。适合长周期、连续迭代的开发任务，既能节省 Token，又能保证工具理解完整项目脉络。

**4. `/config` 交互式配置**

唤起可视化配置界面，支持在线修改模型、主题、权限、超时规则等核心参数。无需退出当前会话，可一键切换模型——例如日常用 DeepSeek 极速编码，遇到复杂架构推理时，临时切换 Claude 旗舰模型，灵活适配不同开发场景。

**5. `/review` 代码审查**

触发专业代码评审机制，让 Claude Code 充当专属 Code Reviewer，对当前项目的新增、修改代码进行全方位校验，覆盖代码正确性、可读性、规范性、安全性、边界场景五大维度。在 iTranslate 项目开发中，每轮功能迭代完成后，都会通过该指令做收尾校验，精准规避人工遗漏的细节问题。

**6. `/cost` 费用统计**

实时查看当前会话的 Token 消耗量与调用费用，是精细化控制开发成本的核心工具。使用 DeepSeek 模型时成本极低，可按需抽查；切换高价 Claude 模型时，建议高频查看，有效控制开销、避免无效消耗。

**7. `/init` 项目配置初始化**

一键生成项目配置文件 `CLAUDE.md` 的初始模板，包含技术栈、目录结构、构建命令等基础信息。工具每次启动会自动读取它来了解项目。**新项目使用 Claude Code 的第一步，就应该执行 /init**——它生成的只是框架，后续需要你持续往里补充编码规范、踩坑经验、高频规则。iTranslate 项目的 CLAUDE.md 从最初几行模板逐渐长到了 200 多行，每条都是开发中踩坑后补上去的。

**8. `/status` 会话状态查询**

实时查看当前会话完整状态，包括已加载的项目文件、活跃的 Agent 任务、运行中的进程等。当不确定 Claude Code 的感知范围、任务执行状态时，通过该指令可快速排查问题、掌握全局进度。

**核心总结**：Slash Commands 本质是**高频操作的标准化快捷键**，彻底解决自然语言重复描述、指令表意模糊的问题，让每一次交互都精准、高效、无冗余。

## 3.2 Agents 并行代理机制

Agents 是 Claude Code 实现**复杂任务拆解、并行开发、独立执行**的核心底层能力，也是其区别于普通 AI 代码工具的关键特性。

简单来说，每一个 Agent 都是一个**独立隔离的 Claude Code 运行实例**，拥有专属上下文窗口、独立的思考与执行链路，可在后台并行运行，完全不干扰主会话和其他任务。你可以将其理解为可随时调度的“专属开发分身”，下发任务后自主执行、完工反馈，无需全程跟进。

Claude Code 内置三类专业化 Agent，分工明确、场景精准，覆盖项目开发全流程：

### 3.2.1 Explore Agent｜代码库探索代理

核心定位：专注代码库检索、逻辑梳理、问题溯源，是项目分析的专属工具。

适用于跨文件逻辑排查、全局代码检索、链路溯源场景。例如批量查找指定函数的所有调用位置、梳理核心功能的完整执行链路、定位隐性 BUG 的触发路径。在 iTranslate 项目开发中，该代理曾在 1 分钟内完成 6 个文件的跨文件检索，精准定位“二次翻译”问题的全部代码路径，效率远超人工检索。

### 3.2.2 Plan Agent｜方案设计代理

核心定位：专注需求拆解、方案设计、架构优化，是项目规划的专属工具。

适用于模糊需求落地、功能迭代升级、架构重构场景。可将零散的需求拆解为可落地的分步执行方案，明确需要修改的文件、新增的接口、兼容的逻辑、迭代的风险点。例如 iTranslate 多语言词典功能迭代中，Plan Agent 自主摒弃原有硬编码逻辑，设计出可扩展的配置化注册表架构，兼顾功能迭代与版本兼容。

### 3.2.3 General-purpose Agent｜通用执行代理

核心定位：全能型执行代理，适配绝大多数代码编写、修改、优化任务，是日常开发的默认执行主体。

代码重构、异常处理补充、样式优化、逻辑迭代等常规开发任务，均由该代理默认执行，覆盖日常 80% 的实操场景。

### 3.2.4 核心优势：并行隔离执行

Agent 机制的最大价值在于**多任务并行 + 上下文隔离**。开发者可同时调度多个子代理，同步推进互不依赖的独立任务，无需串行等待。

例如 iTranslate 开发中，可调度一个代理优化前端弹窗 UI，同时让另一个代理重构后台翻译缓存逻辑，双任务同步推进、互不干扰。且每个代理上下文完全隔离，不会出现任务逻辑混淆、代码错乱的问题。

**使用黄金原则**：全局代码检索、问题溯源用 Explore Agent；需求拆解、架构设计用 Plan Agent；常规编码、代码修改用通用 Agent；多独立任务同步推进用并行子代理。

## 3.3 Skills 可复用工作流系统

Skills 是 Claude Code 的**标准化最佳实践工作流**。区别于单次生效的普通 Prompt，每一个 Skill 都是一套固化、可复用、可强制落地的完整任务流程，将资深开发者的编码规范、调试思路、审查标准转化为工具自动化能力。

### 3.3.1 触发方式

分为主动触发与被动匹配两种模式，灵活适配各类场景：

1. **显式主动调用**：手动输入 `/skill名称`，主动加载对应工作流，强制按照标准化流程执行任务；
2. **智能自动匹配**：Claude Code 自动识别当前任务场景，主动加载适配的 Skill。例如提出“修复代码 BUG”需求时，工具会自动启用标准化调试工作流，按固定步骤排查、复现、修复、验证。

### 3.3.2 Skill 与普通 Prompt 的本质差异

普通 Prompt 仅能传递“做什么”的需求，执行标准依赖工具自主理解，结果随机性强；而 Skill 固化了“怎么做、按什么标准做、做到什么效果”的完整流程，统一执行规范、规避人为疏漏。

简单来说：**Prompt 是单次指令，Skill 是长期规范**，将资深开发者的编程肌肉记忆，转化为可复用、可传承、可自动化的工具能力。

### 3.3.3 Skill 核心价值

- **固化开发纪律**：将“修 Bug 先写测试用例”“改代码必做类型校验”“迭代完成必审查”等优质开发习惯强制落地，避免人工遗忘、随意简化流程；
- **统一团队标准**：团队共用一套 Skill 体系，可实现代码审查、测试、开发的标准统一，杜绝“标准不一、质量参差”的问题；
- **降低新人门槛**：新手无需深耕开发规范与流程，依托标准化 Skill 即可落地专业的开发流程，快速产出高质量代码。

### 3.3.4 新手最优插件组合

Claude Code 插件生态丰富、Skill 数量庞大，新手极易陷入选择内耗。实战证明，无需堆砌插件，两套核心插件即可覆盖**需求澄清→方案设计→编码实现→测试验证→代码审查**全开发链路，完全满足个人项目、中小型项目开发需求。

| 核心插件         | 核心能力与适用场景                                           |
| :--------------- | :----------------------------------------------------------- |
| **superpowers**  | 日常开发主力插件，提供全套基础工作流：需求澄清（brainstorming）、任务规划（writing-plans）、测试驱动开发（tdd）、系统化调试（debugging）、自动化代码审查（code-review） |
| **agent-skills** | 专业级质量校验插件，适配项目收尾关键节点，提供五维度代码审查、标准化测试策略、安全漏洞审计等高阶能力，保障代码质量与项目安全性 |

可选锦上添花插件：**claude-hud**，可在终端生成状态栏，实时展示 Token 用量、运行模型、Agent 状态等信息，直观掌握会话全局状态，非刚需但可优化使用体验。

### 3.3.5 全流程 Skill 实战使用顺序

依托 superpowers 插件，可形成标准化、可复刻的 AI 开发闭环，适配所有常规开发任务：

| 开发阶段     | 对应 Skill 指令  | 核心作用                                                   |
| :----------- | :--------------- | :--------------------------------------------------------- |
| 需求模糊阶段 | `/brainstorming` | 梳理模糊需求，明确功能边界、开发目标与落地细节             |
| 需求明确阶段 | `/writing-plans` | 拆解复杂任务，输出可分步执行的标准化开发清单               |
| 编码开发阶段 | `/tdd`           | 落地测试驱动开发，先写测试、再写实现，保障代码健壮性       |
| 问题调试阶段 | `/debugging`     | 标准化排错，复现问题、缩小范围、定位根因、验证修复         |
| 交付收尾阶段 | `/code-review`   | 五维审查代码，校验正确性、可读性、架构合理性、安全性、性能 |

## 3.4 Hooks 事件自动化机制

Hooks 是 Claude Code 的**事件驱动自动化机制**，核心特性是被动触发、自动执行。与主动调用的 Skills 不同，Hooks 无需人工干预，监听预设事件后自动执行对应操作，彻底解放重复手动操作。

### 3.4.1 核心监听事件

支持全开发流程关键节点监听，覆盖会话生命周期、文件操作、版本控制等场景：

| 监听事件       | 触发时机                             |
| :------------- | :----------------------------------- |
| `PreToolUse`   | 工具执行操作之前（文件编辑、写入等） |
| `PostToolUse`  | 工具执行操作完成之后                 |
| `SessionStart` | 新会话启动初始化时                   |
| `SessionEnd`   | 会话结束、退出时                     |
| `Notification` | 接收系统通知、任务提醒时             |

### 3.4.2 实战落地场景

- **代码修改自动校验**：配置 `PostToolUse` 钩子，代码编辑完成后自动执行 TS 类型校验、语法检测，即时发现报错，避免项目打包时集中爆雷；
- **提交代码自动测试**：监听 Git 提交事件，提交前自动运行项目测试用例，测试不通过则阻止提交，守住项目质量底线；
- **会话初始化自动加载上下文**：会话启动时自动读取项目 README、配置文件，让工具快速熟悉项目现状，减少重复沟通。

### 3.4.3 配置建议

Hooks 属于高级自动化能力，新手无需急于配置。默认配置已能满足绝大多数基础开发场景。建议先熟练掌握基础操作，积累高频重复操作场景后，再针对性配置 1-2 个刚需钩子，避免盲目堆砌无效自动化配置。

## 3.5 标准化日常开发工作流

整合四大核心机制，可形成一套**可复刻、低出错、高效率**的 Claude Code 人机协作开发闭环，适配绝大多数项目迭代场景：

### 步骤1：精准描述需求

摒弃模糊化需求描述，明确功能范围、生效规则、约束条件、使用场景。需求模糊时，优先用 `/brainstorming` 梳理清楚，从源头避免工具开发跑偏。

### 步骤2：工具自主探索代码库

下发需求后，优先让 Claude Code 读取、解析项目相关文件，理解现有架构与代码逻辑，基于现状开发，而非凭空猜测、盲目编写代码。

### 步骤3：审阅优化开发方案

工具编码前会输出完整改造方案、明确修改文件与逻辑。**核心原则：绝不盲从工具方案**。人工审核架构合理性、改动范围、兼容逻辑，及时纠正偏差、规避无效改动。

### 步骤4：精准落地代码实现

优质的 AI 开发是最小范围精准改动，改一行解决问题绝不改动多行。全程观察工具修改逻辑，若出现大范围无关改动，立即叫停并修正指令。

### 步骤5：全维度验证闭环

代码开发完成后，手动执行测试命令、运行项目实操验证。AI 无法 100% 规避 Bug，人工验证是不可或缺的收尾环节，坚决杜绝“写完即结束”的粗放式开发。

### 步骤6：反馈迭代修正

针对验证发现的问题，精准反馈异常现象与预期效果，让工具针对性迭代修复。若连续三轮迭代仍无法解决问题，需回归需求与架构，重新梳理开发思路。

### 三大核心工作原则

1. **小步快跑迭代**：单次会话只开发一个核心功能，多需求拆分多轮迭代，精准定位问题、降低排错成本；
2. **先读后改优先**：强制工具先读取代码、理解现状，再动手开发，杜绝盲目猜测编码；
3. **验证闭环落地**：每轮修改必验证，不堆积问题、不遗留隐患。

### 人机协作核心转变

传统开发的核心是“人工编码、手动排错”，而 AI 辅助开发的核心是**精准提需求、严格审方案、闭环验结果**。开发者的角色从“码字工人”升级为“项目把控者”，这也是高效 AI 编程的核心本质。

## 3.6 高阶实操核心技巧

结合四大机制的实战落地经验，总结五条新手必守的核心技巧，快速拉高 AI 编程效率与代码质量：

1. **新项目必先执行 /init**：`CLAUDE.md` 是工具理解项目的核心载体，无需一次性完善，优先录入技术栈、常用命令，后续持续迭代补充，避免工具重复探索、低效开发；
2. **长对话优先用 /compact 而非 /clear**：连续迭代任务优先压缩上下文、保留核心记忆，仅在完全切换无关任务时清空会话，让工具持续理解项目全貌；
3. **独立任务主动开启并行 Agent**：无依赖的多任务开发，主动告知工具拆分并行子代理，告别串行等待，成倍提升开发效率；
4. **理性看待 Skill，不盲目依赖**：Skill 仅为标准化工具，每次调用后需查看其执行流程，人工判断落地效果，工具是辅助，开发者是最终决策者；
5. **Hooks 宁缺毋滥，只配高频场景**：优先积累开发中的重复操作，仅自动化高频刚需场景，拒绝为了“功能齐全”堆砌无效配置。

## 3.7 本章小结

💡**本章核心知识点汇总**

- **Slash Commands** 是高效交互快捷键，八大核心指令覆盖会话管理、配置、审查、成本统计、项目初始化等全场景，解决重复沟通问题；
