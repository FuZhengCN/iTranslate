## 2.1 前提条件

安装 Claude Code 前，请提前检查并配齐基础运行环境。大部分开发者本地已预装相关工具，建议逐一校验版本，规避后续安装报错。

### Node.js 18+（必备）

Claude Code 基于 Node.js 开发，以 npm 包形式分发，必须依赖 Node.js 18 及以上版本运行。

打开终端，执行版本校验命令：

```bash
node --version
```

输出版本需为 `v18.x.x` 及更高版本。若未安装或版本过低，前往 [nodejs.org](https://nodejs.org) 下载官方最新 LTS 稳定版，默认流程安装即可。安装完成后**重启终端**，让系统环境变量生效。

> **⚠️ 避坑提示**：请勿使用第三方一键安装包、系统预装的老旧 Node 版本。版本不兼容会直接导致 Claude Code 安装失败，常见报错为 `unsupported engine` 或 JS 语法错误，是新手高频问题。

### Git 环境（必备）

Claude Code 依赖 Git 实现文件变更追踪、自动生成提交信息、创建开发分支等核心能力，是项目开发的基础工具。

执行版本校验命令：

```bash
git --version
```

所有 2.x 系列版本均可正常使用。未安装用户可前往 [git-scm.com](https://git-scm.com) 下载对应平台版本，默认安装即可。

### 基础操作与工具要求

无需精通命令行，只需掌握基础终端操作：打开终端、`cd` 目录切换、`ls/dir` 目录查看、`npm install` 依赖安装。

代码编辑器推荐 VS Code（非强制）。Claude Code 纯终端运行，无需 IDE 插件，但需要编辑器查看、修改、调试 AI 生成的代码，VS Code 是当前适配性最佳的选择。

### 跨平台终端适配方案（Windows / macOS / Linux）

本书所有实操命令均为 Unix 风格，原生适配 macOS、Linux 系统。Windows 用户需按需选择适配终端，避免命令不兼容：

| 终端方案     | 推荐度 | 适配说明                                                     |
| :----------- | :----- | :----------------------------------------------------------- |
| **Git Bash** | ⭐⭐⭐⭐⭐  | 随 Git 自动安装，完整兼容 Unix 命令，与本书所有指令 100% 适配，零配置即用 |
| **WSL2**     | ⭐⭐⭐    | 完整 Linux 子系统，功能最全，但需要额外配置，适合有 Linux 开发基础的用户 |

> **严禁使用**系统原生 CMD、PowerShell。二者命令语法、路径分隔符、环境变量配置均与 Unix 终端不同，会导致本书绝大多数命令无法直接运行、代码执行异常。

终端快速自检方法：执行以下命令，输出 `bash` / `zsh` 即为合规终端；空白或报错则需切换至 Git Bash。

```bash
echo $SHELL
```

## 2.2 安装 Claude Code CLI

Claude Code 支持全局一键安装，所有平台通用 npm 命令，安装流程简单高效，耗时 30 秒至 1 分钟（取决于网络状态）。

### 全局安装命令

```bash
npm install -g @anthropic-ai/claude-code
```

### 平台差异化问题解决

Windows（Git Bash）可直接执行上述命令，无权限报错。

macOS / Linux 用户大概率会遇到 `EACCES` 权限报错，核心原因是 npm 全局目录涉及系统权限。

> **最优解决方案**：使用 NVM 管理 Node.js 版本，将运行环境与全局包安装至用户目录，彻底规避系统权限问题，长期开发必备。
>
> **临时解决方案**：命令前加 `sudo` 提权执行，仅适合临时测试，不推荐长期使用。

### 安装校验

安装完成后，执行版本查询命令，输出版本号（`v1.x.x`）即代表安装成功：

```bash
claude --version
```

### 首次启动与认证

终端输入 `claude` 启动交互程序：

```bash
claude
```

首次启动会自动触发两步核心流程：

1. **OAuth 账号认证**：终端输出授权链接，浏览器打开后通过邮箱或谷歌账号注册、授权 Anthropic 账号，授权完成后终端自动接续流程。
2. **进入交互界面**：认证通过后，进入 Claude Code 专属命令行交互窗口，可直接通过自然语言下发开发指令。

### 免费额度说明

Claude Code 官方提供**每日 5 万 Token 免费额度**（以官方最新公告为准），默认调用 Claude Sonnet 模型，完全满足新手学习、小型项目开发、日常调试需求。

> **实操建议**：优先使用官方免费额度熟悉工具交互流程、掌握基础用法，无需开局就配置 DeepSeek。熟练工具逻辑后，再切换低成本模型优化开发成本，循序渐进降低试错门槛。

## 2.3 获取 DeepSeek API Key

官方免费额度有限，且原生 Claude 模型调用成本极高。切换 DeepSeek V4 模型后，开发成本可降低一个数量级，且无需翻墙、支持国内支付，是国内开发者最优选择。整套密钥注册、充值、配置流程仅需 5 分钟。

### 第一步：注册 DeepSeek 开发者账号

打开 [platform.deepseek.com](https://platform.deepseek.com)，通过手机号或邮箱完成注册与身份验证，登录开发者后台。

### 第二步：小额充值

进入后台「费用中心/充值」页面，平台支持**支付宝、微信**国内主流支付方式，最低充值 10 元即可满足长期学习使用。

基于 2026 年 5 月 DeepSeek 永久降价政策，V4 系列模型定价极低，单人开发成本几乎可以忽略不计：

| 模型版本                 | 输入价格（百万 Token） | 输出价格（百万 Token） |
| :----------------------- | :--------------------- | :--------------------- |
| V4-Pro（旗舰复杂推理）   | 3 元                   | 6 元                   |
| V4-Flash（极速日常编码） | 1 元                   | 2 元                   |

实战参考：本书完整 iTranslate 项目（194 次代码提交、全流程调试测试），DeepSeek 总调用成本不足 25 元。仅用于日常学习、简单开发时，10 元余额可支撑一至两个月使用。

### 第三步：生成并保存 API Key

进入后台「API Keys」页面，点击「创建密钥」，系统会生成以 `sk-` 开头的专属密钥。

> **⚠️ 核心重要提醒**：API Key 仅创建瞬间可见，关闭页面后无法二次查看，只能删除重建。生成后立即复制，保存至密码管理器或本地 `.env` 文件，妥善保管、禁止泄露。

### 第四步：密钥有效性校验（可选）

替换下方命令中的密钥，执行接口请求，返回模型 JSON 列表即代表密钥有效、网络通畅：

```bash

  -H "Authorization: Bearer your-key-here"
```

## 2.4 配置 Claude Code 对接 DeepSeek

获取密钥后，可将 Claude Code 后端推理模型从官方 Claude 切换为 DeepSeek V4。提供**图形化一键切换（新手推荐）**和**手动配置（进阶）**两种方案，二选一即可，避免配置覆盖冲突。

### 方式一：CC-Switch 图形化工具（推荐所有新手）

CC-Switch 是国内开发者主流的 Claude Code 模型切换工具，基于 Tauri 2.0 开发，安装包仅 5MB、启动极速、零复杂配置，支持全平台一键切换主流大模型，自动适配认证规则与模型映射，彻底规避手写配置报错。

#### 工具获取与安装

开源地址：[github.com/farion1231/cc-switch](https://github.com/farion1231/cc-switch)

根据系统下载对应版本：Windows 可选 MSI 安装包或绿色版、macOS 解压即用、Linux 支持 Deb 安装包。macOS 首次打开提示「未知开发者」，可在「系统设置-隐私与安全性」中允许运行。

#### 一键配置流程

1. 启动 CC-Switch，新增供应商并选择 **DeepSeek**；
2. 粘贴已保存的 DeepSeek API Key；
3. 按场景完成模型映射（适配 Claude Code 内部角色）；
4. 点击「切换/应用」，自动写入配置，重启终端即可生效。

#### 最优模型映射规则

| Claude Code 内置角色 | 映射 DeepSeek 模型    | 使用场景                                                     |
| :------------------- | :-------------------- | :----------------------------------------------------------- |
| Opus（旗舰推理）     | `deepseek-v4-pro[1m]` | 复杂架构设计、全局重构、高难度 Bug 调试，`[1m]` 开启 100 万超长上下文 |
| Sonnet（主力编码）   | `deepseek-v4-pro`     | 日常功能开发、代码审查、逻辑迭代                             |
| Haiku（轻量快速）    | `deepseek-v4-flash`   | 简单代码生成、批量操作、快速校验任务                         |

> **参数说明**：`[1m]` 后缀代表开启 100 万 Token 超长上下文，适配全项目级开发；无后缀默认 200K 上下文，满足日常轻量开发。CC-Switch 会自动兼容 DeepSeek 思考模式，避免空返回问题。

### 方式二：手动编辑 settings.json（进阶用户）

无需第三方工具时，可直接修改 Claude Code 全局配置文件，精准自定义参数。

#### 配置文件路径

- macOS / Linux：`~/.claude/settings.json`
- Windows（Git Bash 通用）：`C:\Users\用户名\.claude\settings.json`

#### 完整标准配置

创建或修改文件，填入以下内容，替换对应密钥即可：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_AUTH_TOKEN": "sk-你的DeepSeek-API-Key",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "deepseek-v4-pro[1m]",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "deepseek-v4-pro",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "deepseek-v4-flash",
    "API_TIMEOUT_MS": "3000000"
  }
}
```

#### 核心配置关键说明

| 配置项               | 规范值                               | 避坑要点                                                    |
| :------------------- | :----------------------------------- | :---------------------------------------------------------- |
| ANTHROPIC_BASE_URL   | `https://api.deepseek.com/anthropic` | 必须使用 Anthropic 专用兼容端点，不可用 OpenAI 格式的 `/v1` |
| ANTHROPIC_AUTH_TOKEN | DeepSeek 完整 sk- 密钥               | 禁止使用 `ANTHROPIC_API_KEY`，会直接鉴权失败                |
| 模型映射             | 对应 V4 系列模型名                   | 严格匹配命名，避免触发异常思考模式                          |

### 配置有效性验证

配置完成后重启终端、启动 Claude Code，输入测试指令：

```Plain
请用 JavaScript 写一个函数，接收一个字符串，返回它的反转。只输出代码，不解释。
```

正常返回可运行代码即代表配置成功，报错可查阅本章 2.7 节常见问题排查。

> **重要规则**：CC-Switch 图形配置与手动修改 settings.json **二选一**，不可同时使用，否则会出现配置互相覆盖、模型切换失效的问题。

## 2.6 核心配置项速览

Claude Code 配置项繁多，新手无需全盘掌握，优先吃透三个核心配置，即可满足 99% 的开发场景。

### CLAUDE.md（项目级核心配置）

放置于项目根目录，每次启动会话自动读取，用于统一 AI 开发规范、告知工具项目信息，是提升协作效率的关键。可配置内容：

- 项目基础信息（技术栈、框架版本、项目类型）；
- 编码规范（注释语言、变量命名、代码风格）；
- 项目常用命令（构建、测试、启动、打包指令）；
- 高频踩坑规避规则。

该文件无需一次性写完，可随项目迭代持续补充，将重复沟通的规则、踩坑经验固化为自动生效的配置。

### settings.json（全局/项目配置）

分为两级配置：全局配置（`~/.claude/settings.json`）生效于所有项目，项目级配置（`.claude/settings.json`）仅适配当前项目，可覆盖全局参数。核心用途：模型切换、权限管理、生命周期 Hooks 自动化配置。

### 权限管理（安全核心）

默认开启安全校验，工具执行文件操作、系统命令前需人工授权，杜绝误操作风险。可将 `npm test`、`git status` 等安全命令加入白名单，简化确认流程；**严禁将 `rm -rf`、`git push --force` 等高风险命令加入免授权白名单**。

## 2.7 新手高频问题与避坑方案

汇总新手搭建环境的五大高频报错问题，精准定位原因、提供落地解决方案，快速排查修复。

### 坑1：Node.js 版本过低导致安装失败

**报错表现**：执行安装命令提示 `unsupported engine`、JS 语法错误、依赖安装失败。

**解决方案**：通过 `node --version` 校验版本，低于 v18 则通过 NVM 或官网 LTS 版本升级。

### 坑2：代理网络导致接口连接超时

**报错表现**：`fetch failed`、`ECONNREFUSED`、请求超时、模型无响应。

**解决方案**：终端手动配置代理环境变量，适配本地代理端口（Clash 默认 7890）：

```bash
# macOS / Linux / Windows Git Bash 通用
export HTTPS_PROXY=http://127.0.0.1:7890
```

### 坑3：DeepSeek 密钥配置后不生效

**报错表现**：配置完成后提示 `Not logged in`，仍默认调用官方 Claude 模型。

**排查三步法**：

1. 校验认证字段：必须使用 `ANTHROPIC_AUTH_TOKEN`，禁止用 `ANTHROPIC_API_KEY`；
2. 校验接口地址：必须使用 `https://api.deepseek.com/anthropic` 专用端点；
3. 规避配置冲突：手动配置与 CC-Switch 工具不可同时使用，避免互相覆盖。

### 坑4：V4 模型空返回、无响应问题

**报错表现**：配置 V4 模型后，Claude Code 偶尔返回空内容。

**根因**：DeepSeek V4 思考模式下，推理内容存入`reasoning_content` 字段，常规 `content` 字段为空，导致工具读取不到返回结果。

**解决方案**：优先使用 CC-Switch 配置（工具自动适配兼容）；手动配置出现异常时，可通过扩展参数关闭思考模式。

### 坑5：Windows 终端乱码、命令不兼容

**报错表现**：中文乱码、Unix 命令无法识别、路径报错。

**解决方案**：放弃 CMD、PowerShell，统一使用 Git Bash 终端，VS Code 可直接切换内置终端类型。
